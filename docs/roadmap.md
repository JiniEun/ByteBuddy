# 🐧 ByteBuddy - Roadmap

## 🚀 Current Stage (MVP)

### ✅ Implemented

- Electron 기반 데스크탑 앱
- 로컬 Markdown repo 로드
- CS 노트 파싱
- 펭귄 말풍선 (랜덤 질문/요약)
- 클릭 시 상세 내용 표시

---

## 🧩 Phase 1 - Core Stabilization

목표: “기본 앱을 안정적으로 사용 가능하게”

### Tasks

- [ ] Markdown 파싱 구조 개선 (section 분리)
- [ ] 노트 포맷 규칙 정의 (question / summary / answer)
- [ ] UI 안정화 (말풍선 위치, overflow 처리)
- [ ] 에러 처리 (파일 없음, 잘못된 포맷)

---

## 🐧 Phase 2 - UX Enhancement

목표: “펭귄 앱 느낌 강화”

### Tasks

- [ ] 캐릭터 이동 (랜덤 위치)
- [ ] 드래그 이동
- [ ] 애니메이션 (idle / talking)
- [ ] 말풍선 타이밍 조절
- [ ] always-on-top 개선

---

## 📚 Phase 3 - Learning System

목표: “단순 표시 → 학습 도구로 확장”

### Tasks

- [ ] 퀴즈 모드 (질문 → 클릭 시 답변)
- [ ] 태그 기반 필터 (redis, db 등)
- [ ] 즐겨찾기 / 중요 개념 저장
- [ ] 노출 빈도 조절

---

## 🔗 Phase 4 - Repo Integration

목표: “지식 관리 자동화”

### Tasks

- [ ] Git repo 연결
- [ ] 변경 감지 (watcher)
- [ ] 자동 reload
- [ ] multi-repo 지원

---

## 🤖 Phase 5 - AI Integration (Optional)

목표: “지식 확장 및 이해 보조”

### Tasks

- [ ] GPT 기반 설명 확장
- [ ] 쉬운 설명 모드
- [ ] 면접 질문 자동 생성
- [ ] 요약 기능

---

## 📊 Phase 6 - Personalization

목표: “사용자 맞춤 학습”

### Tasks

- [ ] 학습 기록 저장
- [ ] 자주 틀린 개념 강조
- [ ] 반복 학습 시스템 (Spaced Repetition)
- [ ] 사용자 레벨링

---

## 🎮 Phase 7 - Gamification (Optional)

목표: “재미 요소 추가”

### Tasks

- [ ] 경험치 시스템
- [ ] 캐릭터 성장
- [ ] 업적 시스템
- [ ] 일일 학습 목표

---

## 🧭 Direction Principles

개발 시 항상 유지할 기준:

1. 학습을 방해하지 않는다
2. 과한 기능보다 지속성 우선
3. 귀여움은 UX의 핵심 요소
4. 개인 지식 기반을 유지한다

---

## 📌 Final Goal

> “앱을 켜두는 것만으로도  
> 자연스럽게 CS 실력이 쌓이는 환경”

---

## One-line Roadmap

MVP → UX → 학습 시스템 → repo 자동화 → AI → 개인화
